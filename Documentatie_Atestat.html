1. INTRODUCERE

Mini-pbinfo este o aplicație web educațională destinată învățării și practicării programării în limbajul C++. Aplicația este inspirată de platforma populară pbinfo.ro și oferă un mediu interactiv în care utilizatorii pot rezolva probleme de informatică cu evaluare automată în timp real.

Scopul principal al aplicației este de a oferi elevilor un instrument modern și accesibil pentru pregătirea examenului de Bacalaureat la Informatică, în special pentru subiectul III care implică lucrul cu șiruri de caractere.

Obiectivele proiectului:

  • Crearea unei platforme web responsive și intuitive
  • Implementarea unui sistem de evaluare automată a codului C++
  • Oferirea de feedback detaliat pentru fiecare submisie
  • Includerea a 50 de probleme cu șiruri de caractere din programa BAC
  • Suport pentru testare manuală cu input personalizat
  • IntelliSense pentru funcții C++ comune

2. DESCRIEREA APLICAȚIEI

Aplicația funcționează pe modelul client-server, unde:

  • Clientul (Frontend) - rulează în browserul utilizatorului și oferă interfața grafică
  • Serverul (Backend) - procesează cererile, compilează și evaluează codul trimis

Fluxul principal de utilizare:

  1. Utilizatorul accesează aplicația în browser la adresa http://localhost:3000
  2. Selectează o problemă din lista disponibilă
  3. Citește enunțul problemei (cerință, format intrare/ieșire, exemple)
  4. Scrie soluția în editorul de cod integrat
  5. Opțional: testează codul cu input personalizat
  6. Trimite soluția pentru evaluare
  7. Primește verdictul (Accepted, Wrong Answer, Compile Error, Time Limit Exceeded)

Aplicația salvează automat în browserul utilizatorului:

  • Problema selectată curent
  • Input-ul personalizat pentru testare

  • Numărul de încercări greșite (pentru deblocarea hints)

7. SISTEMUL DE EVALUARE (JUDGE)

7.1 Procesul de Evaluare

┌────────────────────────────────────────────────────────────┐
│                    JUDGE SYSTEM                             │
├────────────────────────────────────────────────────────────┤
│                                                             │
│  1. VALIDARE COD                                           │
│     • Verificare cod gol                                   │
│     • Verificare dimensiune maximă (100KB)                 │
│     • Verificare include-uri interzise                     │
│       (windows.h, thread, filesystem)                      │
│                                                             │
│  2. COMPILARE                                              │
│     Comandă: g++ -o program source.cpp -std=c++17 -O2     │
│     Timeout: 10 secunde                                    │
│     Eroare → Compile Error + mesaj explicativ             │
│                                                             │
│  3. EXECUȚIE PE TESTE                                      │
│     Pentru fiecare test (1 → testsCount):                 │
│     • Citește input din {n}.in                            │
│     • Rulează programul cu timeout                        │
│     • Compară output cu {n}.out                           │
│     • Time Limit: problema.timeLimit (default 2000ms)     │
│                                                             │
│  4. VERDICT FINAL                                          │
│     • Accepted - toate testele trecute                    │
│     • Wrong Answer - output incorect                      │
│     • Time Limit Exceeded - depășire timp                 │
│     • Compile Error - eroare la compilare                 │
│     • Runtime Error - crash la execuție                   │
│                                                             │
└────────────────────────────────────────────────────────────┘

7.2 Exemple de Verdicte

Verdict Cod Descriere
Accepted (AC) ✓ Soluția a trecut toate testele
Wrong Answer (WA) ✗ Output incorect pe cel puțin un test
Time Limit Exceeded (TLE) ⏱ Programul a depășit limita de timp
Compile Error (CE) ⚠ Eroare la compilare
Runtime Error (RE) 💥 Programul a crash-uit (ex: segmentation fault)

7.3 Cod Sursă Judge (Simplificat)

async function runJudge(problemId, code) {
    // 1. Încarcă problema
    const problem = loadProblem(problemId);
    
    // 2. Validare cod
    if (!code.trim()) return { verdict: 'Compile Error' };
    
    // 3. Compilare
    try {
        execSync(`g++ -o program source.cpp -std=c++17 -O2`);
    } catch (err) {
        return { verdict: 'Compile Error', message: err.stderr };
    }
    
    // 4. Rulare pe teste
    let passedTests = 0;
    for (let i = 1; i <= problem.testsCount; i++) {
        const input = readFile(`tests/${i}.in`);
        const expected = readFile(`tests/${i}.out`);
        
        const output = runProgram(input, problem.timeLimit);
        
        if (output === expected) passedTests++;
        else return { verdict: 'Wrong Answer', passedTests };
    }
    
    return { verdict: 'Accepted', passedTests };
}

8. BAZA DE DATE CU PROBLEME

8.1 Format Problemă (JSON)

{
  "id": 1,
  "title": "Lungimea șirului",
  "category": "Șiruri de caractere",
  "timeLimit": 1000,
  "testsCount": 5,
  "description": "Se citește un cuvânt...\n\n**Cerință:** Afișează lungimea...",
  "tips": [
    "Folosește funcția strlen(s)",
    "Parcurge șirul până la '\\0'"
  ]
}

8.2 Categorii de Probleme

Aplicația conține 50 de probleme organizate pe categorii:

Nr. Categorie Exemple de probleme
1-10 Operații de bază Lungime șir, numărare vocale/consoane, inversare, palindrom
11-20 Numărări și statistici Suma cifrelor, frecvența caracterelor, caractere distincte
21-30 Manipulare șiruri Prefix, sufix, rotație, subșir, inserare, ștergere
31-40 Prelucrare cuvinte Numărare cuvinte, cel mai lung cuvânt, sortare, cifrul Caesar
41-50 Operații complexe Verificare ordine, litere alternante, subșiruri de vocale

8.3 Structura Testelor

Fiecare problemă are 5 teste:

  • 1.in - input pentru testul 1
  • 1.out - output așteptat pentru testul 1
  • ... (până la 5.in, 5.out)

Exemplu test pentru problema "Lungimea șirului":

Fișier Conținut
1.in informatica
1.out 11
