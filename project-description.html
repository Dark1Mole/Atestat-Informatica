1. Argument

Învățarea programării în C++ necesită practică constantă și feedback imediat asupra soluțiilor scrise. Majoritatea platformelor de entrenamet (cum ar fi pbinfo.ro, codeforces.com, etc.) sunt accesibile doar online și necesită internet continuu. Problemele pe care le conțin sunt adesea prea complexe pentru începători.

Mini-pbinfo este un sistem local de evaluare automată a soluțiilor C++, proiectat special pentru elevii care se pregătesc pentru examenul de atestat. Sistemul permite:

  • Rularea și testarea soluțiilor pe calculator local, fără acces la internet
  • Feedback imediat cu verdicte clare (Accepted, Wrong Answer, Time Limit Exceeded, Compile Error)
  • Probleme graduale, de la intro până la niveluri mai complexe
  • Interfață user-friendly cu editor de cod și descrierea problemelor în Markdown
  • Distribuție sub formă de aplicație web modernă, ușor de desfășurat pe orice mașină

Scopul acestui proiect este să ofere o sursă de practică accesibilă și eficientă, ajutând elevii să-și îmbunătățească abilități în programare C++ și să se pregătească cu succes pentru examenul de atestat.

2. Noțiuni Teoretice

2.1. Concepte Fundamentale

Un sistem de evaluare automată (automated judging system) este o platformă software care compară automat rezultatele unui program cu ieșirile așteptate. Componente principale:

  • Compilator: Transformă codul sursă C++ în executabil
  • Executor: Rulează programul cu input predefinit
  • Comparator: Compară output-ul produs cu cel așteptat
  • Evaluator: Determină verdictul (Accepted, Wrong Answer, etc.)
  • Manager de timp: Impune limite de timp pentru executare (timeout)

2.2. Arhitectura Sistemului

Mini-pbinfo utilizează o arhitectură client-server:

Frontend (Client)

Aplicație React cu editor de cod Monaco, interfață responsivă, și comunicație cu backend prin HTTP REST.

Backend (Server)

Server Node.js/Express care expune API-uri pentru probleme, compilare, execuție și evaluare.

Judge (Evaluator)

Modul separat care compilează codul C++ și compară rezultatele pe bază de test cases.

Storage

Probleme și test cases stocate local în fișiere JSON și .in/.out.

2.4. Fluxul de Evaluare

  1. Trimitere cod: Elevul scrie C++ și trimite prin /api/submit
  2. Validare: Se verifică dacă codul îndeplinește restricții (mărime, includuri interzise)
  3. Compilare: Se compilează cu g++ -std=c++17 -O2, cu timeout 10s
  4. Execuție: Pentru fiecare test case, se rulează programa cu input predefinit (timeout pe test)
  5. Comparare: Output-ul se normalizează și se compară cu așteptat (ignoră spații, \r\n)
  6. Verdict: Se determină verdictul final și se returnează cu detalii
  7. Logging: Se salvează în log verdicul și testele trecute

4. Exemple de Probleme

Categoria: Introducere

Problema 1: Suma a două numere
Se citesc doi întregi a și b. Să se afișeze suma lor.
Problema 2: Verificare număr par
Se citește un întreg. Să se afișeze "Par" sau "Impar".

Categoria: Cicluri

Problema 4: Factorial
Se citește n. Să se calculeze și afișeze n!
Problema 5: Verificare număr prim
Se citește n. Să se verifice dacă este prim.

Categoria: Șiruri de caractere

Eliminare litere mici
Se citește un cuvânt. Să se afișeze cu toate literele mici șterse.

Sistemul conține 50+ probleme organizate pe categorii progressive, de la probleme simple de introducere până la algoritmi mai complecși.

5. Deployment și Publicare

Sistemul este configurat pentru deployment pe platforme cloud populare:

  • Frontend: Desfășurat pe Vercel (https://atestat-informatica.vercel.app)
  • Backend: Desfășurat pe Render (https://atestat-informatica.onrender.com)
  • Versionare: Cod stocat pe GitHub și auto-sincronizat cu deploys

Configurația permite:

  • Deploy automat la fiecare push pe branch main
  • Environment variables pentru API base URL
  • CORS corect configurat pentru comunicare cross-origin
  • Build cache și optimizări de performanță

6. Concluzii

Mini-pbinfo este o aplicație web completă care demonstrează cunoștințe solide în:

  • Programare C++: Compilare, execuție și testare de programe C++
  • Backend Development: Server Node.js/Express cu API REST
  • Frontend Development: Aplicație React modernă cu editor de cod
  • DevOps: Deployment pe Vercel și Render, CI/CD cu GitHub
  • UX/UI Design: Interfață responsivă și ușor de utilizat
  • Gestiune de proiecte: Arhitectură modulară și cod bine organizat

Sistemul este funcțional și gata de utilizare, oferind:

  • Sursă de practică accesibilă pentru elevii care se pregătesc pentru atestat
  • Feedback imediat și precis asupra soluțiilor
  • Platformă extensibilă pentru adăugarea de noi probleme
  • Cod open-source și ușor de dezvoltat ulterior

Proiectul poate fi utilizat și îmbunătățit continuu cu: teste unitare, analiză de performanță, suport pentru alte limbaje de programare, și funcționalități sociale (clasamente, comentarii).
